Declaring Variable

VARIABES
Variables are named values and can store any type of JavaScript value.

Here’s how to declare a variable:
EXAMPLE:

let x = "Hello universe";

And here’s what’s happening in the example above:
- let is the keyword that tells JavaScript you’re declaring a variable.
- x is the name of that variable.
- = is the operator that tells JavaScript a value is coming up next.
- "Hello universe" is the value for the variable to store.


In JavaScript, there are three ways to declare variables:
using the var, let, and const keywords. Each keyword has its own characteristics and scope rules.

Here are examples of each type of variable declaration:
let declaration:
The let keyword was introduced in ECMAScript 6 (ES6) and provides block-scoping, allowing variables to be limited to
the nearest enclosing block (commonly a pair of curly braces {}).
Variables declared with let can be reassigned later within their scope.
EXAMPLE:
let x = 10;
console.log(x);

Output: 10

const declaration:
The const keyword also appeared in ECMAScript 6 and is used to declare constants that cannot be reassigned.
Like let, const is also block-scoped.

const x = 10;
EXAMPLE:
console.log(x);

Output: 10
x = 20; // Error: Assignment to constant variable

var declaration:
The var keyword has been traditionally used for variable declaration in JavaScript, but it has some quirks and scoping issues.
Variables declared with var are function-scoped, meaning they are accessible within the function they are declared in or at the global scope if declared outside of any function.
EXAMPLE:

// Function-scoped variable
function example() {
var x = 10;
console.log(x);
Output: 10
}
console.log(x);
// Error: x is not defined (not accessible outside the function)


Overall, it is recommended to use let and const instead of var to avoid scoping issues and to have more predictable code behavior.
The choice between let and const depends on whether you need to reassign the variable or if you want to ensure its immutability.
